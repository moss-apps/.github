# Contributing to Moss Apps

Thanks for contributing. Here is how to get started.

## Before You Start

- Search existing issues and pull requests to avoid duplicates.
- For significant changes, open an issue first to discuss your approach.

## Pull Request Process

1. Fork the repository and create a branch from `main`.
2. Follow the existing code style and naming conventions in the project.
3. Keep pull requests focused. One concern per PR.
4. Test your changes thoroughly. If the project has a test suite, make sure it passes.
5. Update documentation if your change affects public APIs or user-facing behavior.

## Code Style

- Match the conventions already present in the codebase.
- Write clear, self-documenting code. Comments should explain intent, not mechanics.
- No trailing whitespace, consistent indentation.

## Reporting Bugs

Include:
- App version and environment (OS, device model)
- Steps to reproduce
- Expected vs. actual behavior
- Logs or screenshots if applicable

Use the repository's issue tracker. If a security vulnerability is involved, see [SECURITY.md](SECURITY.md).

## Feature Requests

We are open to ideas. Open an issue describing:
- The problem you want to solve
- How your proposed feature would work
- Why it belongs in the project
